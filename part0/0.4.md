sequenceDiagram

  participant server

# 0.4 — New note (traditional web app)

```mermaid
sequenceDiagram
  participant browser
  participant server

  browser->>server: GET /notes (document)
  server-->>browser: HTML (contains form + script)
  browser->>server: GET /main.css
  server-->>browser: main.css
  browser->>server: GET /main.js
  server-->>browser: main.js
  Note right of browser: user fills form and clicks Save
  browser->>server: POST /new_note (form data)
  server-->>browser: 302 Redirect -> Location: /notes
  browser->>server: GET /notes
  server-->>browser: HTML (updated)
  browser->>server: GET /data.json
  server-->>browser: JSON data
```

Comments:
- The POST uses form encoding; server responds with a redirect (302) causing the browser to reload `/notes`.
- Reload triggers fetching CSS, JS and the JSON data again.
